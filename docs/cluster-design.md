## 多实例部署

Message Service 支持水平扩展，每个实例通过以下参数区分：

| 环境变量 | 说明 | 实例1 | 实例2 |
|----------|------|-------|-------|
| `SNOWFLAKE_WORKER_ID` | Snowflake 工作节点 ID | 1 | 2 |
| `SERVER_ID` | 实例标识 | msg-1 | msg-2 |

## 在线状态管理

- 每个实例在用户连接时将 `serverId` 写入 Redis：`collab:msg:online:{userId} = msg-1`
- TTL 120 秒，心跳刷新
- 通过 `serverId` 可以知道用户连接在哪个实例上
- Redis 不可用时，回退到本地 `WsSessionManager` 判断

## 消息可靠性

1. **先持久化再广播**：消息先写入 MySQL，再通过 Pub/Sub 广播
2. **异步持久化**：使用独立线程池（CompletableFuture + ThreadPoolExecutor），不阻塞 WebSocket IO 线程
3. **离线消息**：接收者离线时，消息已持久化，可通过 `/session/offline` 接口拉取
4. **去重保障**：Redis SETNX + 24h TTL 防止重复投递
